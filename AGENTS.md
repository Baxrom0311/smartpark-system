# Agent rules for this repository

All AI coding agents must follow these rules:

1. Do not expose secrets, tokens, credentials, or private files.
2. Prefer small, reviewable changes.
3. Run the configured verification command after meaningful edits.
4. Do not introduce new frameworks or services without a reason tied to the brief.
5. Keep generated code maintainable: clear names, simple structure, useful errors.
6. When reviewing, focus on correctness, security, reliability, tests, and missing acceptance criteria.
7. When implementing, do not stop at planning; edit files and verify.
